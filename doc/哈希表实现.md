# 哈希表实现（项目一）

本文档说明项目一中哈希表的核心实现思路：使用 **拉链法（Separate Chaining）** 处理冲突。

## 为什么使用哈希表

考试报名系统以“考号”为唯一键，常见操作为按考号查找、插入与删除。哈希表在平均情况下能提供接近 O(1) 的访问性能，适合此类场景。

## 冲突处理：拉链法

同一个哈希桶中可能出现多个考号映射到同一索引，此时：

- 每个桶保存一个单链表头指针
- 新节点可以采用头插法插入链表
- 查找/删除在对应桶的链表内线性扫描

## 哈希函数与桶数

实现中桶数选用质数（例如 997），有助于减少某些输入模式下的聚集。

哈希函数需要满足：

- 输出范围：`[0, bucket_count-1]`
- 尽量均匀分布（同一输入集合下）

## 关键操作（逻辑说明）

- **查找**：计算桶索引 → 遍历该桶链表 → 找到考号返回
- **插入**：先查重（可选）→ 头插到桶链表 → 计数 +1
- **删除**：计算桶索引 → 双指针（`prev/current`）遍历 → 摘链并释放节点 → 计数 -1
- **修改**：先查找节点 → 更新非键字段（保持考号不变）

## 复杂度（简要）

- 平均：查找/插入/删除接近 O(1)
- 最坏：退化为 O(N)（当大量键落在同一桶时）
