# DFS 回溯算法（迷宫求解）

本仓库的迷宫项目（P3）使用深度优先搜索（DFS）结合回溯思想来寻找路径。

## DFS 的基本思想

DFS 的核心是“尽可能向深处走”：

1. 从起点开始
2. 选择一个可行的方向前进
3. 若能继续前进则不断深入
4. 若遇到死路则回退到上一步，尝试其他方向

## 回溯与状态恢复

在迷宫中，回溯需要解决两个问题：

- **避免循环**：用 `visited[][]` 标记已经到达过的位置
- **撤销选择**：当某条路径走不通时，需要撤销当前步骤带来的状态变化

常见做法：

- 进入一个格子时：`visited[x][y] = true`
- 从该格子回溯返回时：`visited[x][y] = false`（允许其他路径再次经过）

## 路径保存

实现通常会在搜索过程中记录路径（例如用数组模拟栈）：

- 进入新格子：把坐标压入路径栈
- 回溯时：弹出路径栈顶
- 到达终点：路径栈即为一条可行解

## 复杂度（简要）

设迷宫可通行格子数为 `V`，每个格子最多有 4 条边：

- 时间复杂度：O(V + E)（网格图中 E 约为 4V）
- 空间复杂度：O(V)（`visited` 与递归栈/路径栈）
