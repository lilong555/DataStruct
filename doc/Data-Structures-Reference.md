# 数据结构参考

本仓库 6 个项目涉及的数据结构如下，便于复习与对照代码理解。

## 1. 数组（Array）

用途：保存顺序数据、随机访问。  
典型项目：P3（迷宫矩阵）、P8（邻接矩阵）、P10（待排序数组）。

特点：

- 随机访问 O(1)
- 插入/删除（非尾部）通常为 O(N)

## 2. 链表（Linked List）

用途：动态存储、插入删除方便。  
典型项目：P1（哈希冲突拉链）。

特点：

- 插入/删除（已定位节点）O(1)
- 查找 O(N)

## 3. 哈希表（Hash Table）

用途：按键快速查找。  
典型项目：P1（考号→考生信息）。

特点：

- 平均查找/插入/删除接近 O(1)
- 最坏情况可能退化为 O(N)（冲突严重）

## 4. 栈与递归栈（Stack）

用途：回溯、路径记录。  
典型项目：P3（DFS 搜索路径时的递归栈/路径栈）。

## 5. 队列（Queue）

用途：按先来先服务顺序处理数据。  
典型项目：P5（A/B 两个窗口队列）。

## 6. 堆（Heap）

用途：动态维护“最小值/最大值”。  
典型项目：P7（最小堆反复取最小两段）。

特点：

- 取堆顶 O(1)
- 插入/删除堆顶 O(log N)

## 7. 图（Graph）

用途：描述顶点之间的连接关系与边权。  
典型项目：P8（电网建设造价）。

实现方式：

- 邻接矩阵：适合顶点数较小、图较稠密的情况
