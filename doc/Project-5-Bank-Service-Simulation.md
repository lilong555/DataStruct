# 项目五：银行业务模拟

本项目模拟银行双窗口（A、B）办理业务的完成顺序。

## 题目规则（要点）

- 顾客编号为奇数 → 到 A 窗口
- 顾客编号为偶数 → 到 B 窗口
- A 窗口速度是 B 的 2 倍（可理解为：A 每处理 1 个单位时间，B 需要 2 个单位时间）
- 当两窗口同时完成业务时，优先输出 A 窗口顾客

## 输入输出

输入为一行正整数：

```
N id1 id2 ... idN
```

输出为一行顾客编号，表示按“业务完成顺序”的输出结果，编号间以空格分隔，末尾无多余空格。

## 实现思路（简要）

1. 先将顾客按奇偶分配到 A、B 两个队列
2. 使用两个“剩余服务时间”变量模拟窗口进度：
   - A 处理时间为 1
   - B 处理时间为 2
3. 每次比较两个窗口的剩余时间：
   - 谁更小，谁先完成并输出该队列队首
   - 若相等，先输出 A 再输出 B
