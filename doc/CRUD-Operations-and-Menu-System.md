# 增删改查（CRUD）与菜单系统（项目一）

项目一采用“初始化录入 + 菜单循环”的交互方式，将对考生信息的操作拆分为多个独立功能。

## 菜单设计

典型流程：

1. 启动程序后先录入 N 位考生信息，建立初始数据集
2. 进入循环菜单，用户反复选择功能并输入参数
3. 用户选择退出后程序结束

菜单选项（示例）：

- `1` 插入
- `2` 删除
- `3` 查找
- `4` 修改
- `5` 统计/显示
- `0` 退出

## CRUD 操作说明

### 1）插入（Create）

- 录入新考生信息
- 以考号为唯一键，必要时先做“查重”
- 插入成功后可选择输出当前列表

### 2）查找（Read）

- 用户输入考号
- 在哈希表中定位桶并遍历链表
- 找到则输出该考生信息，否则提示未找到

### 3）修改（Update）

- 先按考号定位目标节点
- 考号作为唯一键通常不允许修改
- 只更新姓名/性别/年龄/类别等字段

### 4）删除（Delete）

- 用户输入考号
- 在对应桶链表内删除匹配节点并释放内存
- 删除成功后更新总人数并输出结果（可选）

## 输入合法性与交互鲁棒性

为避免无效输入导致程序异常，建议（并在实现中体现）：

- 对数字输入（考号/年龄/人数）做 `cin` 失败判断并清空缓冲区
- 对性别字段限制为 `男/女`
- 对字符串长度做基本限制
