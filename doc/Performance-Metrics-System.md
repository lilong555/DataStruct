# 性能统计系统（项目十）

项目十为每种排序算法统计三类指标：时间、比较次数、交换/移动次数。

## 统计字段

在实现中通常会为每次运行维护一份统计结构体（示例字段）：

- `algorithm_name`：算法名称
- `compare_count`：比较次数
- `swap_count`：交换/移动次数
- `time_elapsed_ms`：运行时间（毫秒）

## 比较次数的统计口径

当代码出现“关键字比较”（例如 `a > b`、`a < b`、`a <= b`）时计数。  
注意：对于基数排序等非比较类排序，比较次数可能为 0 或者只统计了少量辅助比较（如寻找最大值）。

## 交换/移动次数的统计口径

不同算法的数据移动方式不同：

- 交换（swap）：通常计为 1 次
- 移动（赋值搬移）：也可计入 `swap_count`，用于粗略衡量数据搬移开销

因此该指标主要用于同一实现下的相对对比，并不等价于真实 CPU 指令数。

## 时间统计说明

实现使用 `clock()` 获取 CPU 计时器差值并换算为毫秒。  
在不同机器/系统上，绝对时间可能不同，但同机对比仍具有参考意义。

## 建议的对比方式

- 固定随机数生成方式与范围，保证可重复
- 对多种数据规模（如 100、1000、10000、100000）分别测试
- 观察 O(N²) 与 O(N log N) 算法在规模扩大时的增长趋势
